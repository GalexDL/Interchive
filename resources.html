Notte's guide for the Interchive

This section is to explain each button and menus present in the Interchive!

First off we have the 3 main buttons on the main menu, Main Campign, Halidom Tales, and Event Stories. Main Campaign contains the main Dragalia story and Event Stories contains nothing at this moment. Halidom Tales contains castle stories, adventuerer/dragon stories, and creator stories

On the top-left corner you can see a picture of Euden! This is where you can type your name! This is important because by default the name is "none"

On the top-right corner you can see a picture of gears. This is the options menu, you can change a various amount of things but its best to leave most things as is since it can mess up the viewing expirence

In the middle we have the load button, this button allows you to load a previous save point! This works differently in the creator version. Also specifc to the creator version there is a test button that allows you to quickly test stories.

In the Halidom Tales Menu you can type or scroll to find a specifc story.



Getting Started

No coding expirence is needed but this is still a learning process made easy! Please dont be frighten to learn how to create, its easy I promise.

Open the renpy-8.0.3-sdk folder and open renpy.exe, I recommend pinning this on to your desktop or taskbar.
This is the launcher for ren’py and there's a video if you want to customize it to how you like, if you wish to do so. Tutorial/Question are built into ren’py and you cant remove them and aren't created by me. By default Interchive should be selected anyways.

  • Open Directory: 'Game' opens your file explorer into the Interchive Files. Here lies all of the RPY files and Images. Don't click any py files, you will regret it.

  • Edit File: 'Open Project' should ask you to select a default code editor. I highly recommend Atom, because in my opinion it visually looks the best, but you can choose at your discretion.

    • After you select it you should see the game folder or 10 .rpy files. Open the game folder if you cant see the rpy files. A few of them will not matter to you for simplicity sake. This should wrap up the outline of the launcher and setting up renpy, the structure section goes into indepth detail explaining the strcuture of files.

    Structure

    All of the game files are present under 'game', first thing you will see is a bunch of folders containing UI Images, Audio Files, Backgrounds, and Portraits. If you plan on adding your own images I suggest making a new folder! You will also see the rpy and py files. DO NOT CLICK THE PY FILES, itll be your biggest mistake.

    Going now into the structure of the RPY files, where you will be typing up your story and etc.

  • Charactername: Contains character names
  • Gui: Usage isn't important but contains some stuff relating to the UI
  • Options: Usage isn't needed but contains some overall function code
  • Script: Usage isn't needed
  • Functions: Contains animations and some other attributes
  • Name: Contains the function for the Prince Name
  • Chapter1: Usage isn't needed
  • Test: Main file for testing your story quickly
  • Character: Contains all the information about Image Portraits and it's no where close to done because there are like 700 characters and still needs manual testing to fix expressions (help)
  • Debug: Usage isn't needed
  • Background: Contains background image data
  • Cutscene: Contains video based cutscene data


    • Tutorial #1

    I’ll go in depth in other sections about documentation/meaning in other sections, but let's get right into it. Other sections will be important as well, this section is mainly to get the feel of it. Saves work differently in this developer mode since you are adding things, so don't try to make saves because it'll be inaccurate. Before we begin please launch the game and input your name if you havent.

    Open 'test.rpy' we will be creating the story here, I already layed out some basic groundwork.
    Label is the name of the story, this name is used so a button can call a specific story. This is very important because you do not want to potentially overlap other stories, I would avoid using anything too basic for universal usage purposes. Return will always be at the end of a script, so it can take you back to the main menu.

    test.rpy
                            
label test:

    #Key starting Values!
    stop music
    scene bgcastle1
    show black
    with fade
    ################
    show prince

    prince "test"
    show textboxlog

    return

    On line 9 it says show prince, this makes the Euden appear and you can make him disappear later in the script with “hide prince”. You can change Euden’s facial expression with show prince eye_4 mouth_4. All sprites have various amounts of mouths/eyes and you can refer to the DLPortrait Viewer for reference. Also to note currently all portraits are not supported, and facial emotion placement may be incorrect. This google sheet is my current reference for characters implemented.

    Moving on to line 11, This line makes the textbox place the players name and the text “test”. You can type almost anything within those quotation marks, except for a few things thats used for customization which I will not be discussing in this section except for two of them. These two are “\n” and “{nw}”, currently the textbox barrier isn't complete so \n is very useful to make words go to the next line. {nw} purpose is to make a line automatically continue, this is mainly for intros fading out or a panic scene. Each line should contain text for one scene, so dont write an essay…

    test.rpy
                            
    show prince eye_4 mouth_4

    prince "Testing \nTesting line 2"
    
    show textboxlog
    
    prince "Its funny this is called no wait{nw}"
    prince "I waited"
    
    show textboxlog

    return

    We are going to look at charactername.rpy, this file contains the names of characters. This is very important because you absolutely can not put any random name. You must define as stylized in this file define “coding name” = Character("In game name", kind=character_base). For example if I am adding the name Alex I would put define alex = Character("Alex", kind=character_base). Now in test.rpy I can put alex “I am here”

    charactername.rpy
                            
define aurelius = Character("Aurelius", kind=character_base)
define zethia = Character("Zethia", kind=character_base)
define notte = Character("Notte", kind=character_base)
define Mysterious = Character("???", kind=character_base)
define skylaa = Character("Skyla", kind=character_base)

define (codename) = Character("Character Name", kind=character_base)

define alexx = Character("Alex", kind=character_base)

    test.rpy
                            
    alexx "Oh wait I dont look like alex..."

    Lastly for Tutorial #1 is action movements. So far there are 4 technical movements, Bounce, Moving Left/Right/Center and one transitionary action fadein.

  • Bounce makes the sprite jump

  • moveleft makes the sprite slide to the left

  • moveright makes the sprite slide to the right

  • movecenter makes the sprite slide to the center

  • fadein is a transitionary action that makes a character appear and dissapear.

  • movefadein is used when the charcter is already shown and using another action such as moveleft

    • Examples of thier usage is below, As a reminder at fadein is not needed when hiding. To clearfy movefadein existence, its due to fadein counting showing again as hiding hopefully this is fixed eventually.

    test.rpy
                            
    show skyla at moveright at fadein
    skylaa "I can move around crazy, not that crazy but \nI can just imagine it"
    show skyla at movecenter, movefadein
    skylaa "All the information you need about the \nDragalia Interchive is in the here"     
    hide skylaa
    
    return
